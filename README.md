# openclaw-gtm-skills

[![License: AGPL v3](https://img.shields.io/badge/License-AGPL_v3-blue.svg)](https://www.gnu.org/licenses/agpl-3.0)

An [OpenClaw](https://openclaw.ai) plugin + skill collection for full-stack B2B outbound GTM. Turns domains into researched, scored, segmented, and outreach-ready prospects. A Clay.com alternative you run yourself.

Inspired by and partially based on [extruct-ai/gtm-skills](https://github.com/extruct-ai/gtm-skills). See [ATTRIBUTION.md](ATTRIBUTION.md).

---

## What's Inside

**Standalone skills** (can be used independently):
- `context-building` ‚Äî build/update a GTM context file (ICP, voice, proof, campaigns, DNC list)
- `hypothesis-building` ‚Äî generate testable pain hypotheses from ICP + win cases
- `email-prompt-building` ‚Äî craft self-contained email prompt templates for campaigns
- `email-generation` ‚Äî run prompts per-contact from CSV ‚Üí generate personalized emails
- `list-segmentation` ‚Äî segment + tier companies by hypothesis fit + data richness
- `gtm-hypothesis-scorer` ‚Äî RICE scoring for all GTM hypotheses in your DB

**Plugin pipeline** (at `plugins/beton-gtm/`):
- `gtm-intake` ‚Üí `gtm-prefilter` ‚Üí `gtm-research` ‚Üí `gtm-sales-org` ‚Üí `gtm-signals` ‚Üí `gtm-segment` ‚Üí `gtm-outreach` ‚Üí `gtm-deck`
- Full end-to-end: domain list ‚Üí researched companies ‚Üí fit scores ‚Üí outreach emails ‚Üí sales decks

## Requirements

- [OpenClaw](https://openclaw.ai) installed and running
- [Supabase](https://supabase.com) project (for the pipeline database)
- [Firecrawl](https://github.com/mendableai/firecrawl) ‚Äî self-hosted or [cloud](https://firecrawl.dev)
- Attio API key (optional ‚Äî only for CRM sync in `gtm-segment`)
- [seqd](https://github.com/getbeton/seqd) (optional ‚Äî for outreach enrollment in `gtm-outreach`)

## Standalone Skills Setup

Copy skills to your workspace:

```bash
cp -r skills/* ~/.openclaw/workspace/skills/
```

Each skill has a `SKILL.md` explaining when to use it and what it does. Example workflow:

1. **context-building** ‚Äî capture product, ICP, voice, proof points
2. **hypothesis-building** ‚Äî define pain hypotheses for a vertical
3. **email-prompt-building** ‚Äî build a campaign-specific email prompt
4. **email-generation** ‚Äî run the prompt against a CSV of contacts

See individual skill READMEs for details.

## Plugin Pipeline Setup

### 1. Apply the database schema

```bash
supabase db push --workdir ./plugins/beton-gtm/supabase --password <your-db-password>
```

### 2. Configure the plugin

Edit `plugins/beton-gtm/openclaw.plugin.json` and fill in your values:

```json
{
  "config": {
    "supabaseUrl": "https://yourproject.supabase.co",
    "supabaseKey": "your-service-role-key",
    "attioApiKey": "optional",
    "firecrawlUrl": "http://localhost:3002",
    "seqdApiUrl": "optional"
  }
}
```

### 3. Install in OpenClaw

Copy the plugin directory to your workspace:

```bash
cp -r plugins/beton-gtm ~/.openclaw/workspace/plugins/
```

### 4. Run the pipeline

Start with a CSV of domains:

```bash
cd plugins/beton-gtm/scripts
python3 run_intake.py --input domains.csv
python3 run_prefilter.py --limit 50000 --workers 20
python3 run_research_combined.py --limit 1000 --workers 5
python3 run_segment.py
```

## Database Schema

Full Supabase schema in `plugins/beton-gtm/supabase/migrations/`. Tables:

- `companies` ‚Äî core entity, tracks `research_status` through the pipeline
- `company_classification` ‚Äî GTM motion, vertical, pricing model, B2B/SaaS flags
- `company_firmographics` ‚Äî headcount, funding stage
- `company_tech_stack` ‚Äî detected tools (CRM, analytics, etc.)
- `company_sales_org` ‚Äî sales/revops headcount, hiring signals
- `company_social` ‚Äî LinkedIn URL
- `segments` ‚Äî your ICP segments with pain hypotheses + value props
- `company_segments` ‚Äî company ‚Üî segment scoring (fit_score, fit_tier)
- `value_hypotheses` ‚Äî segment-level pain hypotheses (generated by hypothesis-building)
- `gtm_experiments` ‚Äî HADI cycle tracking for outreach experiments
- `signals` ‚Äî buying signals per company
- `contacts` ‚Äî people found for outreach
- `outreach` ‚Äî drafted sequences, review status
- `results` ‚Äî outcomes (replied, bounced, etc.)

---

## Database Conventions

- Flat relational tables with foreign keys ‚Äî no JSONB for structured data
- All schema changes via `supabase migration new` ‚Äî never raw SQL
- Migration files are the source of truth
