What changed

v1 → v2

v1 (12 steps, 30 scripts)

  • Manual script-by-script execution
  • 5 duplicated Apollo wrappers
  • 5 duplicated Firecrawl wrappers
  • 3 different Claude classifiers
  • No email verification gate
  • No DNC list enforcement
  • No cross-campaign dedup
  • Claude-based scoring (variable)

v2 (7 steps, 1 CLI)

  • Agent-first: skills + Python modules
  • 1 Apollo client, 1 Firecrawl client
  • Apollo org enrichment before crawl
  • Role-based signals feed scoring
  • Dedicated email verification gate
  • DNC + cross-campaign dedup
  • Hard reject catch-all emails
  • Deterministic scoring (pure rules)

Architecture

7-Step Pipeline

Agent-first design: each step is a skill (SKILL.md) that Claude Code / OpenClaw executes, backed by a Python module. One CLI entry point: python beton.py run domain.com. Every step reads/writes Supabase. Resume from any point via per-company pipeline_status.

Step 3

Research

Three phases, Apollo-first. Get structured data for free before spending on Firecrawl + Claude.

  • Phase A: Apollo org enrichment FREE — employee count, industry, revenue, country, city, founded year, keywords. Filter out companies with <10 or >5000 employees
  • Phase B: Firecrawl + Claude — map + scrape 12+ pages (homepage, about, pricing, product, careers, blog, case studies, integrations, docs, changelog). Single Claude call outputs classification, sales org, tech stack, signals, agent goals
  • Phase C: Apollo people search FREE — search all 4 thread title sets. Store found roles in company_leads (no email reveal yet). These roles feed directly into scoring
One Firecrawl crawl + one Claude call + two free Apollo calls replaces what was 3 separate steps with 3 Claude calls. Store every field — we're paying for the calls anyway.

Step 5

Contacts: Reveal + Verify

People were already found in step 3 (Phase C). This step only reveals emails and runs them through verification. No gap fill — if a thread has 0 valid contacts, skip it.

  • Reveal 1 CREDIT — Apollo people/match on top lead per thread from company_leads
  • Role-based filter — Hard reject: info@, support@, noreply@, admin@, sales@, marketing@, hr@, billing@
  • Email verification NEW — Dedicated provider (MillionVerifier / ZeroBounce / Reoon). Hard reject: invalid, disposable, spam trap, catch-all
  • DNC check NEW — Check dnc_list table for email or domain match
  • Cross-campaign dedup NEW — Skip if same email contacted in last 90 days
Credits only spent on T1/T2 companies (scored in step 4). Catch-all emails are hard rejected — they damage sender reputation.

Step 6

Sequence Drafting

  • Per contact — 5-step email sequence personalized to thread, role, company vertical, and pain hypothesis
  • 4 thread templates map to 4 angles: churn signals (strategy), at-risk users (CS), expansion signals (revenue), intent data gap (sales)
  • Vertical mapping — 10+ verticals with specific user_type, data_action, churn/expansion examples
  • Writing rules enforced — All lowercase, 60 word max, no banned words, concrete CTAs, breakup mentions open source
  • Pre-send validation — Haiku checks that company facts, signals, and contact data are accurate before approving

Campaign Design

4-Thread Choreography

Each company gets up to 4 parallel threads targeting different decision-makers. Threads are staggered so it looks like coordinated outreach. Leads found in step 3, revealed in step 5.

ThreadTargetSubject Pattern
StrategyCSO, VP Analytics{company}'s {user_type} churn signals
CSDir CS, VP Retentionsaving {company}'s at-risk {user_type}
RevenueCRO, VP Growth{company}'s expansion signals
SalesVP Sales, Dir Sales{company}'s pipeline signals beyond intent
Per company: up to 20 emails over ~10 days across 4 threads. Empty threads are skipped, not gap-filled.

New in v2

Email Validation Gate

Every email passes through a 4-layer validation chain before any sequence is generated. No exceptions.

LayerCheckAction
1Role-based email (info@, support@, noreply@)Hard reject
2Email verification (invalid, disposable, spam trap)Hard reject
3Catch-all domainHard reject
4DNC list or contacted in last 90 daysHard reject

Results cached in email_validations table — never re-verify the same email. DNC list managed via CLI: python beton.py dnc add user@example.com

Catch-all emails are rejected with zero exceptions. They damage sender reputation and inflate bounce rates.
